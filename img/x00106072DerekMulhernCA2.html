I am an IT Management Student

For my second CA, I was required to build an Interactive Portfolio Website using Twitter Bootstrap. Having never used Bootstrap before it was a challenge, but a very enjoyable exercise. The template the writer choose to use was Musim Bocor (Original Template)

  • Law of Pragnanz - I certainly did keep my website very simple. I saw so many students very abstract background wallpapers, where i decided to choose a simple green tinted timber floor.
  • Law of Continuity - The arrangment of my website is completely facilitated to the user, easy to read and straight to the point. Everything that is required within a portfolio is included within the website
  • Law of Similarity - The sections follow a sequence, white then a light grey to allow the user acknowledge a new section of the website. A title for each section with animation included such as fade in or boucing titles
  • Law of Focal Point - I incorporated this law by using different, stand out colors for the buttons, tooltips and the project icons
  • Law of Proximity - Having three columns in education and projects allows the user acknowledge one with the other. The modals were kept simple and incorporated simple requirements
  • Law of Isomorphic Correspondence - Sticking with popular conventions of simple websites such as a navigation bar at the top, enabling smooth scrolling and a method a fast method to give a user the ability to return to the top of the screen were all taken into consideration
  • Law of Figure/Ground - The colors chosen for this website really fall under this law. Everything on the page is very well spaced and very legible on all types of devices such as i-Phone, HTC One, i-Pad, 13" Laptop and finally on 27" montitors
  • Law of Common Fate - This law was not really necessary regarding my simple portfolio website.

Test Cases - Unfixed Bugs / Fixed Bugs

Unfixed Bugs
1) When a user is viewing the website on a mobile device, they can slightly drag the screen to the left and right
2) Interacting with the website using Miscrosoft Edge browser, animinations do not work as intended. For example the Education banners are suppose to rotate (from left to right) onto the screen as they do in Chrome, however in this browser, they slide in from the right.

Fixed Bugs
1) The drop-down menu would not work properly on mobile devices. I had to add CSS properties after debugging the website for many hours
2) To make sure all components were aligned on all devices, I had to add buttons to the project section, i tried to use the "hidden" property, however this would not work. My solution, use disabled buttons
3) The scholar gylphicon would not work. After some investigations I discovered that the index page required a more up to date link to a bootstrap page. The one from the template was out of date
4) Google maps would not work initially. If you use CSS on a separate file it will not work, however if you incorporate inline styling, it eventually worked.
5) Sometimes the nav bar drop-down menu, covered the about me section. It was because the z-index was set wrong on the original template as that did not have this option incorporated. After a few days of debuggins and some conversations with many students I discovered this solution and fixed the problem
